Tip Keamanan 1: Cegah SQL Injection!

Saat pembeli mencari part lewat filter, inputan pencarian rentan disisipi query jahat jika kamu koding sembarangan menggunakan teknik sambung string (String Concatenation).

Cara SALAH & BAHAYA (SQL Vulnerable) $search = $_GET['search'];
$query = "SELECT * FROM tb_products WHERE name LIKE '%$search%'";

Hacker bisa memasukkan input ' UNION SELECT username, password FROM users -- untuk mencuri password admin!

Cara BENAR & AMAN (Prepared Statements) // Menggunakan PDO di PHP
$stmt = $pdo->prepare("SELECT * FROM tb_products WHERE name LIKE :search");
$stmt->execute(['search' => "%" . $search . "%"]);
$results = $stmt->fetchAll();

Sistem akan memperlakukan input hacker murni sebagai teks pencarian, bukan kode perintah SQL!

Analogi Otomotif Koding

Database = Rak & Laci Gudang

Tanpa indeks database, sistem harus membaca data dari baris pertama sampai terakhir. Ini seperti mekanik yang harus membongkar seluruh kardus di bengkel hanya untuk mencari 1 busi!

Validasi Backend = Saringan Oli

Validasi input di frontend (HTML) itu mudah dilewati oleh bot atau curl. Backend harus memvalidasi ulang (mengecek tipe data, panjang teks, ukuran file) layaknya filter oli yang memisahkan kotoran besi dari mesin utama.

API Backend = Kabel Kelistrikan (Wiring)

Menghubungkan tombol "Beli" di layar depan (Frontend) dengan pengurangan stok di Gudang Database (Backend) membutuhkan kabel data berupa RESTful API (JSON).
